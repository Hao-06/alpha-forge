## 一、亮点挖掘

**一句话**：AlphaForge 是面向全球加密交易者的「多 Agent 决策驾驶舱」——3 个 LLM Agent 协同分析实时行情，从 5 个内置策略中**自主选择 / 微调 / 组合**最适配当前 regime 的方案，输出可解释的交易信号。

### 创新点 1：多 Agent 角色分工，多模型协作
不是一个 LLM 一锤子买卖，而是**模型选型与角色严格匹配**：
- **DeepSeek-R1**（强推理）担任 **行情判断 Agent**——输出 5 种 regime 的概率分布，承认不确定性
- **Claude 4.5**（综合 + 跟随规则）担任 **策略选择 Agent**——权重分配 + 微调建议
- **GPT-5**（保守 + 合规感）担任 **风险审核 Agent**——一票否决权

这一组合不是堆模型，而是把**每个模型放在它最擅长的位置**——这是我们在 API 调用质量维度上的核心论据。

### 创新点 2：策略库 × Agent 自进化
传统策略机器人："写死一个策略，行情变了就崩"。
AlphaForge："策略是子集，Agent 在子集上做**实时再分配**"。

策略库（5 个）：
| 策略 | 适配 regime | 信号来源 |
|---|---|---|
| 📈 动量趋势 | trending_up / down | MA7/MA25 金叉死叉 + 偏离度 |
| 🔁 均值回归 | ranging / high_vol | RSI 极值反转 |
| 🪜 网格交易 | ranging | 区间中枢位置 |
| 🚀 突破策略 | trending + 高量 | 20 期高低 + 量比 |
| 💧 DCA 定投 | panic / 熊市末 | 24h 跌幅放大权重 |

策略选择 Agent 看到当前 regime 后，**动态分配权重 + 给出微调建议**——这就是"主动升级的智能体"的最小可演示版本。

### 创新点 3：可解释 + 可审计
- 每一次决策的 R1 思维链、Agent 输出、调用模型、token 用量**全部落盘**
- UI 的「开发者面板」直接展示 GMI API 调用日志，评委一目了然
- 决策输出 JSON 格式，下游可对接任意执行端

### 差异化（vs 常规出海工具）
| 维度 | 常规加密策略工具 | AlphaForge |
|---|---|---|
| 决策方式 | 单一策略硬编码 | **多 Agent 协作 + 策略库再分配** |
| 模型使用 | 单 LLM | **多模型角色化分工** |
| 可解释性 | 黑盒输出 | **三层推理可视化** |
| 风险控制 | 后置止损 | **前置审核 Agent + 否决权** |

## 四、产品详解

### 4.1 产品定位
为全球加密交易者打造的、24/7 实时多 Agent 决策助手——**降低策略选择疲劳，提升跨指标分析效率，把"看图操作"升级为"AI 策略官辅助决策"**。

### 4.2 目标海外用户群体
- **核心**：北美 / 东南亚 / 欧洲的散户与小型基金加密交易者
- **特征**：24/7 关注 BTC/ETH/SOL 等主流币；策略碎片化；缺乏跨指标系统化分析能力；英文是主操作语言

### 4.3 应用场景
- 早盘 / 夜盘行情突变时，5 秒得到 Agent 解读
- 在多个策略间犹豫时，让 Agent 用当前 regime 给出权重组合
- 把 Agent 输出 JSON 作为交易系统下游输入

### 4.4 技术架构
```
┌─────────────────────────────────────────────────────────────┐
│         Streamlit 决策驾驶舱（深炭灰 × 古金色 终端风）         │
│  ┌──────────┬──────────┬──────────┐                         │
│  │📊 Regime │🎯Strategy│🛡️  Risk  │  → 🧠 决策官 → JSON 信号  │
│  └────┬─────┴────┬─────┴────┬─────┘                         │
│       │          │          │                                │
│       ▼          ▼          ▼                                │
│   GMI Cloud Inference Engine API（OpenAI-compatible）        │
│   DeepSeek-R1  Claude-4.5    GPT-5                           │
│                                                               │
│   ↑ 实时行情：CCXT × Binance/OKX × BTC/ETH/SOL                │
└─────────────────────────────────────────────────────────────┘
```

### 4.5 核心功能
| 功能 | 实现要点 |
|---|---|
| 实时行情 | CCXT + Binance/OKX，K 线/RSI/MA/波动率指标 |
| Regime 识别 | DeepSeek-R1 输出 5 种 regime 概率分布 |
| 策略选择 | Claude 4.5 从 5 策略中分权重 + 给出微调建议 |
| 风险审核 | GPT-5 输出 verdict（approve/warn/veto）+ 风险评分 1-10 |
| 综合决策 | Σ weight × signal_score，风险否决直接 hold |
| 调用日志 | 全 GMI API 调用记录（agent / model / 延时 / tokens / 状态） |

### 4.6 未来规划
1. **真实 RL 自进化**：把"Agent 微调策略参数"从 LLM 提示升级为基于历史回测的真 RL
2. **多市场扩展**：加密 → 美股 → 港股，统一 Agent 框架
3. **执行层对接**：Binance / OKX API 实盘下单（需合规许可）
4. **个性化**：用户风险偏好画像 + Agent 个性化调权
5. **多语言界面**：英文 / 日文 / 韩文（贴合目标用户分布）

## 五、GMI Cloud 使用说明

### 5.1 接入方式
GMI Cloud Inference Engine 提供 **OpenAI 兼容 API**，本项目直接使用官方 `openai` Python SDK，仅替换 `base_url` 即可：

```python
from openai import OpenAI

client = OpenAI(
    api_key=settings.llm.api_key,          # 从 GMI 控制台获取
    base_url="https://api.gmi-serving.com/v1",
)
```

完整客户端实现见 [`alphaforge/llm/gmi_client.py`](../alphaforge/llm/gmi_client.py)。

### 5.2 调用的模型清单
| Agent | 模型 ID | 选型理由 |
|---|---|---|
| 📊 行情判断 | `deepseek-ai/DeepSeek-R1` | 强推理，适合输出概率分布 + 思维链 |
| 🎯 策略选择 | `anthropic/claude-sonnet-4.5` | 综合推理 + 跟随规则（权重和=1、单策略≤0.5） |
| 🛡️ 风险审核 | `openai/gpt-5` | 保守、合规感强，适合做风险否决 |

### 5.3 链路设计
```
       ┌─────────────────────────────────────────────────┐
       │  调用 GMI Inference Engine（统一 base_url）      │
       └─────────┬────────────┬─────────────┬────────────┘
                 ↓            ↓             ↓
        DeepSeek-R1      Claude 4.5       GPT-5
        (response_      (response_       (response_
         format=json)    format=json)     format=json)
                 │            │             │
                 ▼            ▼             ▼
         RegimeAgent → StrategyAgent → RiskAgent → DecisionOfficer (本地)
                            │
                            └─→ 每次调用记入 CallLog
                                  ↓
                            UI 开发者面板可视化
```

### 5.4 实际效果
- **单次决策耗时**：约 6-10 秒（三 Agent 串行 + R1 思维链）
- **稳定性**：所有调用都通过统一客户端，错误自动记入日志且不让 UI 崩溃
- **可观测性**：开发者面板逐次记录 prompt 预览、延时、token 用量、状态
- **降级能力**：未配置 Key 时进入 mock 模式，UI 仍可演示流程（应对路演网络风险）
